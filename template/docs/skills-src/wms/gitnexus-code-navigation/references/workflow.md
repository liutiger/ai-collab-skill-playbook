# Workflow

## 标准顺序

1. `status`：确认索引是否可用
2. `query`：按业务词、异常词、接口名找流程和模块
3. `context`：看候选符号的调用者、被调用者、参与流程
4. `impact`：改公共符号、核心方法、接口定义前先评估 blast radius
5. 只有图谱无法覆盖时，再做局部 `rg`

## 回退机制

- 第 1 轮失败：重写查询词，不改搜索面
- 第 2 轮失败：回退到“我要确认的到底是入口、主编排、分支还是副作用点”
- 第 3 轮仍失败：总结已知证据和缺口，不继续盲搜

## 反模式

- `save|insert|create|update` 混搜
- `.*` 拼接多个方法名
- 未确认模块前直接扫 `**/*.java`
- 同一大文件随机打开多个区间碰运气
