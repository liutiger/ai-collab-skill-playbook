# Governance Orchestrator Output Contract

输出必须包含以下 5 段：

## 1. 总控判定

- 主场景
- 次场景（可选）
- 当前模式
- 当前阶段
- 叠加的方法（可选）

## 2. 当前轮次目标

- 本轮只做什么
- 本轮不做什么

## 3. 核心分析 / 执行结果

- 关键证据、关键改动或关键结论
- 风险与边界
- 需要沉淀的文档

## 4. 待确认

若存在 hard gate：

- 一次性列出所有必须确认项
- 明确用户确认后将继续的下一步

## 5. 收尾检查

- 是否还有新增补充要求
- 是否允许写完成标记
- 是否需要继续检查文档
