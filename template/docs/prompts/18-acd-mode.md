# 提示词 18 — ACD 策略包

> 对应 Strategy Pack：`acd-mode`
> 本文件是 ACD 的兼容参考入口；默认由 `/wms-orchestrator` 自动选择是否采用该策略。

## 目标

先收敛边界、架构方案、接口契约、风险和回滚，再决定是否进入实现。

## 默认顺序

1. 明确目标、边界、非目标
2. 列出约束条件与受影响模块
3. 给出候选方案及取舍
4. 明确风险、回滚和验证方式
5. 停在待确认；确认后再转入实现或 TDD

## 约束

- 人工确认前，不要直接进入实现
- 不要把取舍明显不同的方案混成一句话
- 若任务已经足够清楚且验收可测试，可切到 `tdd-mode`
- 若证据还不够，先切到 `aod-mode`

## 输出协议

```markdown
## ACD 策略
- 架构目标与边界：
- 受影响模块 / 契约：
- 候选方案：
- 取舍与风险：
- 回滚与验证：
- 待确认项：
```
