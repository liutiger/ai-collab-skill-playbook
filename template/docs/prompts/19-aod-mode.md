# 提示词 19 — AOD 策略包

> 对应 Strategy Pack：`aod-mode`
> 本文件是 AOD 的兼容参考入口；默认由 `/wms-orchestrator` 自动选择是否采用该策略。

## 目标

先做证据盘点、链路观察、假设收敛和止血建议，再决定是否需要代码修复或方案确认。

## 默认顺序

1. 盘点已知证据与缺失证据
2. 梳理时间线、影响范围和关键链路
3. 给出止血建议和验证动作
4. 收敛假设，不把猜测写成结论
5. 证据足够后，再决定进入 ACD 或 TDD

## 约束

- 不要一上来直接写代码
- 不要把日志猜测当成根因
- 若需要确认真实执行链路，可叠加 `link-trace-and-curation`
- 证据已足够且要讨论方案时，切到 `acd-mode`

## 输出协议

```markdown
## AOD 策略
- 已知证据：
- 缺失证据：
- 时间线 / 影响范围：
- 当前假设：
- 下一步验证动作：
- 是否需要切换策略：
```
