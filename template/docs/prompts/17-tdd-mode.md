# 提示词 17 — TDD 策略包

> 对应 Strategy Pack：`tdd-mode`
> 本文件是 TDD 的兼容参考入口；默认由 `/wms-orchestrator` 自动选择是否采用该策略。

## 目标

先把验收标准转成测试点、失败信号和回归断言，再做最小实现。

## 默认顺序

1. 明确可验证的验收标准
2. 先列要新增 / 修改的测试点或断言
3. 说明当前应出现的失败信号
4. 给出最小实现路径
5. 完成后进入评测门禁

## 约束

- 不要先写实现、后补测试
- 不要在验收标准还模糊时强行进入 TDD
- 若出现公共接口、数据库结构或跨服务取舍，切回 `acd-mode`
- 若证据不足、现象不清，切回 `aod-mode`

## 输出协议

```markdown
## TDD 策略
- 验收标准：
- 测试点 / 断言：
- 失败信号：
- 最小实现路径：
- 回归验证：
```
