# Workflow

## 标准阶段

1. Planner：澄清目标、范围、验收标准、关键假设
2. Context Analyst：完成 GitNexus 定位和影响面识别
3. Solution Architect：确定最小改动路径、风险、回滚、验证
4. Implementer：按边界实现改动
5. Reviewer & Tester：做架构、自检、验证准备、残留风险说明
6. Evaluator / Gatekeeper：执行或明确记录评测门禁，给出交付结论
7. Documenter：写 README、对话记录、沉淀与完成标记

## 进入条件

- 任务目标已明确
- 已确认当前场景
- 高风险任务已完成方案确认

## 退出条件

- 改动与评测门禁已完成
- 风险已透明化
- 文档沉淀已完成
